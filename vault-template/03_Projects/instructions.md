# 03_Projects — Instructions

## Purpose
Concrete projects with a tangible deliverable — an app, tool, workflow, or creative output.

**This is NOT for learning tracks.** Learning n8n or Claude Code belongs in `05_Learning/`.

A project lives here when it has:
- A specific goal with a finish line
- Something being built or produced
- Progress that can be tracked week to week

## Structure

Each project subfolder:
```
03_Projects/[Project-Name]/
├── README.md      → What is this? Goal, why it matters, current status
├── progress.md    → Session logs, blockers, next steps
└── logs/          → (optional) detailed session notes
```

## Learning vs Project — The Distinction

| `05_Learning/` | `03_Projects/` |
|----------------|----------------|
| Learning n8n concepts | Building a specific n8n workflow for X |
| Learning Claude Code skills | Building a Claude Code tool that does Y |
| Ongoing, no finish line | Has a deliverable and a done state |
| Notes and insights | Output and progress |

## Project Lifecycle
1. **Created** — You decide to build something specific
2. **Active** — Session logs, blockers, weekly updates
3. **Completed** — Final summary written, learnings moved to `05_Learning/`
4. **Archived** — Move to `_archive/` subfolder
