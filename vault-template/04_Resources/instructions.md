# 04_Resources — Instructions

## Purpose
Reference shelf for materials you may revisit: docs, books, links, frameworks, and reusable snippets.

## What Belongs Here
- Tool references (API docs, cheatsheets)
- Book/article notes meant as reference, not active learning logs
- Curated links and source lists
- Templates you reuse across projects

## What Does NOT Belong Here
- Raw thoughts (use `02_Inbox/`)
- Ongoing concept learning (use `05_Learning/`)
- Project execution logs (use `03_Projects/[name]/`)
- Daily session tracking (use `01_Daily/`)

## Suggested Structure
```
04_Resources/
├── docs/
├── books/
├── links/
└── templates/
```

Use any structure that stays easy to browse.

## How Claude Uses This
- Pulls references when answering "how do I..." questions
- Suggests relevant resources during `/weekly` synthesis
- Can add references here when you explicitly ask to save source material

## Style Note
Keep entries concise and scannable. If a resource becomes a real skill/insight, promote it to `05_Learning/` with your own explanation.
